1.  Executive Summary

This proposal outlines a research project that asks whether a deep generative neural network can independently rediscover the statistical behavior of a classical physical system — the two-dimensional Ising model — using nothing but example images. No equations, no Hamiltonians, and no statistical mechanics will be shown to the network. It will only ever see binary spin-lattice images.

The original simulation code was written in Fortran and supplied by the faculty. Since this implementation will be carried out without access to a Linux/Fortran toolchain, the simulator will be faithfully reimplemented in Python, preserving the exact same physics, the same Metropolis Monte Carlo update rule, and the same parameters (L, equilibrium_cycle, measurement_cycle). This Python simulator becomes the data generation engine for the entire project.

Once a dataset of spin configurations is generated across a sweep of temperatures, a Generative Adversarial Network (GAN) will be trained to reproduce these configurations. The project's central deliverable is not a GAN that produces visually convincing images — it is quantitative proof, using real statistical-physics metrics (magnetization, energy, spin correlation, and cluster-size distributions), that the GAN has internalized the same physical laws that govern the original simulation, including the critical phase transition.

Core Research Question

Can a neural network discover the statistical laws of a physical system purely from image examples, without being told any physics?



This question sits in a growing area of scientific machine learning, and the techniques developed here — binary image generation, distribution matching, and physics-based validation of generative output — transfer directly to medical image synthesis, molecular generation, materials discovery, and climate modeling.

2.  Scientific Background

2.1  The Two-Dimensional Ising Model

The Ising model is one of the most studied models in statistical physics. A material is represented as an L × L grid, where every cell holds a 'spin' value of either +1 (spin up) or −1 (spin down). For this project, L = 128, giving a lattice of 128 × 128 = 16,384 spins. Every lattice configuration can be rendered directly as a binary image, where +1 maps to a white pixel and −1 maps to a black pixel.

Spins interact with their nearest neighbors through the Ising Hamiltonian:

H = -J * Σ(i,j neighbors)  s_i * s_j

where J is the coupling constant (J > 0 for a ferromagnet) and the sum runs over all nearest-neighbor pairs. This single equation is the entire physical law governing the system — and critically, the GAN trained in this project will never see it.

2.2  The Role of Temperature

Temperature T controls how much thermal noise disrupts spin alignment. The behavior of the system falls into three regimes:

Regime

Physical Behavior

Visual Signature

Low temperature
(T < Tc)

Spins align into a single dominant direction — an ordered, ferromagnetic phase

Large uniform regions of one color; very few isolated flipped spins

Critical temperature
(T ≈ 2.269 J/kB)

The phase transition. Order and disorder compete at every length scale

Fractal-like clusters of all sizes — the most visually and statistically rich regime

High temperature
(T > Tc)

Thermal noise dominates; spins flip randomly — a disordered, paramagnetic phase

Salt-and-pepper noise pattern with no large-scale structure



2.3  Why the Critical Point Is the Real Benchmark

Near the critical temperature, the correlation length of the system diverges and clusters appear at every scale simultaneously — a property known as scale invariance. This is the signature of a continuous phase transition, and it is mathematically the hardest regime to reproduce, because there is no single dominant length scale for a generative model to latch onto. A GAN that only reproduces convincing images at very low or very high temperature has learned something close to memorization. A GAN that reproduces the correct cluster statistics at the critical point has learned something closer to the actual physics. For this reason, performance near Tc is treated as the most important checkpoint in evaluation (Section 6).

3.  Reimplementing the Supplied Fortran Code in Python

The professor's existing simulator is written in Fortran and is designed to run in a Linux environment. Since this project will be developed without Linux or a Fortran compiler available, the simulation will be faithfully reimplemented in Python — preserving identical physics, identical algorithmic steps, and identical parameters — so that the resulting datasets are scientifically equivalent to what the original Fortran code would have produced.

3.1  What the Fortran Code Currently Does

Based on the parameters and structure of the supplied code, the simulator implements a standard 2D Ising model evolved with the Metropolis Monte Carlo algorithm. The key parameters identified in the code are:

3.2  Algorithmic Steps to Reproduce

The Python reimplementation will reproduce the following Metropolis Monte Carlo procedure, sweep for sweep:

Initialize an L × L lattice with random +1/−1 spins.

For each Monte Carlo sweep, visit every lattice site (or select sites at random) and compute the energy change ΔE that would result from flipping that spin, using the four nearest neighbors with periodic boundary conditions.

Accept the flip with probability min(1, exp(−ΔE / kBT)) — the Metropolis acceptance rule.

Repeat for equilibrium_cycle sweeps without recording anything, so the system reaches thermal equilibrium at temperature T.

After equilibration, continue sweeping and save a snapshot of the full lattice every fixed number of sweeps, for measurement_cycle total saved configurations.

Repeat the entire procedure independently for each temperature in the sweep (Section 4.2).

3.3  Python Translation Strategy

The translation will proceed in three stages of increasing performance, so correctness is verified before speed is optimized:

Stage A — Pure NumPy reference implementation: a straightforward, readable vectorized version using NumPy array operations. This version is the 'ground truth' against which faster versions are checked, since it is the easiest to verify by hand against known Ising model results (e.g. critical temperature, expected magnetization curve).

Stage B — Numba-accelerated version: the same algorithm compiled with Numba's @njit decorator for near-C performance. This is necessary because Monte Carlo spin updates are inherently sequential and do not vectorize well in pure NumPy; Numba allows a fast, simple loop-based implementation instead.

Stage C (if needed) — Checkerboard/vectorized updates: splitting the lattice into a checkerboard pattern so that all spins of one color can be updated simultaneously without conflicting with their neighbors, enabling GPU or full-array vectorization for the largest lattice sizes (L = 256).

3.4  Validation Against Known Physics

Before any GAN training begins, the Python simulator itself must be validated, independent of the original Fortran code, using well-established theoretical results for the 2D Ising model:

The theoretical critical temperature Tc = 2 / ln(1 + √2) ≈ 2.269 (Onsager's exact solution) must appear as a sharp feature in the simulated magnetization-vs-temperature curve.

Magnetization should approach ±1 at low T and approach 0 at high T, with a smooth but increasingly sharp transition as L grows.

Specific heat and susceptibility (computed from energy and magnetization fluctuations) should show a peak at Tc.

Visual inspection: generated lattice images should visibly match the three regimes described in Section 2.2.

Why this matters

If the Python simulator does not reproduce known Ising model physics, none of the GAN evaluation later in this project would be meaningful — the 'ground truth' dataset itself would be flawed. This validation step is non-negotiable and is scheduled as its own milestone in Section 7.

5.  Generative Model Architecture

5.1  Model Selection Rationale

Binary spin images are simpler than natural images (only two pixel intensities, strong local correlation structure, periodic boundary symmetry), which makes convolutional GANs a well-suited and computationally efficient choice. Three architectures will be implemented in increasing order of sophistication:

Stage

Model

Purpose

Baseline

DCGAN

Establish a working pipeline; fast to train and debug

Primary

WGAN-GP

Wasserstein loss with gradient penalty for stable, mode-collapse-resistant training

Extension

Conditional WGAN-GP

Temperature-conditioned generation (Section 5.4)



5.2  Generator Architecture

Input: latent noise vector z ~ N(0, I), dimension 100

4 transposed-convolution blocks, each followed by BatchNorm and ReLU

Progressive upsampling: 4×4 → 16×16 → 64×64 → 128×128

Final layer: Tanh activation, producing pixel values in [−1, +1] (mapped back to {−1, +1} spins via thresholding for evaluation)

5.3  Discriminator / Critic Architecture

Input: 128×128 single-channel image (real or generated)

4 convolutional blocks with LeakyReLU and spectral normalization (replacing BatchNorm, which is incompatible with the WGAN-GP gradient penalty)

Output: a single scalar realism score (no sigmoid — required for Wasserstein loss)

5.4  Temperature Conditioning (Planned Extension)

Once the unconditional WGAN-GP is validated, the architecture will be extended into a conditional GAN by injecting the temperature value into both the generator and critic — concatenated into the generator's latent vector and added as an extra channel in the critic's input. This allows a single trained model to generate configurations at any requested temperature, including temperatures that were sparsely sampled or entirely unseen during training, directly testing whether the network has learned a smooth, physically meaningful representation of the phase diagram rather than memorizing isolated temperature buckets.

6.  Physics-Based Evaluation Framework

Visual plausibility is not sufficient evidence of success. The central scientific claim of this project — that the GAN has learned the underlying statistical physics — must be supported by quantitative agreement between physics-simulated and GAN-generated configurations across four independent metrics.

6.1  Magnetization

M = (1/N) * Σ s_i          (N = total number of spins)

Magnetization measures net spin alignment. The full distribution of M across many generated samples — not just its average — will be compared against the physics simulation at each temperature, since matching only the mean would hide failures in variance or higher-order behavior.

6.2  Energy

E = -J * Σ(i,j neighbors) s_i * s_j

The energy distribution is a direct probe of how well local spin-spin interactions are reproduced, independent of global alignment. Energy histograms from GAN samples will be overlaid against simulation histograms at matching temperatures.

6.3  Spin-Spin Correlation Function

C(r) = < s_i * s_j >   for all pairs at separation r

This measures how spin alignment decays with distance. Near Tc, this decay follows a power law rather than an exponential one — a hallmark of criticality. Reproducing the correct correlation function shape, especially near Tc, is considered the single most rigorous test in this project.

6.4  Cluster Size Distribution

Using connected-component labeling, the sizes of all same-sign spin clusters will be extracted from both real and generated images. At the critical temperature, this distribution should follow a power law (scale-free clusters of all sizes); away from Tc, it should be dominated by either one giant cluster (ordered phase) or many small clusters (disordered phase).

6.5  Comparison & Reporting Method

For every metric, results will be reported in three complementary forms:

Overlaid histograms / kernel density plots — physics simulation vs. GAN, per temperature

Metric-vs-temperature curves — to check that the GAN reproduces the same phase transition shape as the physics simulation, not just isolated matching points

A single scalar discrepancy score (Wasserstein distance) between real and generated distributions — enabling an at-a-glance, quantitative summary table across all temperatures and all four metrics

Definition of Success

The project is considered scientifically successful if the GAN-generated distributions for magnetization, energy, and correlation function fall within a small, pre-defined Wasserstein-distance tolerance of the physics simulation at every tested temperature — including, critically, near Tc.

The project is structured into seven phases over an estimated seven-week period. Each phase has a concrete deliverable, so progress can be checked against the original Fortran-derived specification at every stage.

	Phase 4  Baseline DCGAN	Week 4

	Phase 7  Final Evaluation, Report & Presentation	Week 7

Prepare a presentation deck highlighting the critical-point results as the centerpiece finding
